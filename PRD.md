# PRD: Asgard - Unified Agent Integration Platform

**文档版本：** 1.0  
**创建日期：** 2025-02-08  
**产品定位：** 面向多维场景的 Agent 集成网关，通过标准协议将各类定制 Agent 接入 IDE、写作工具、创意软件及 SaaS 应用。覆盖开发、写作、创意、分析等全领域。

---

## 1. 核心定义 (Core Concepts)

| 概念 | 说明 |
|------|------|
| **Asgard Key** | 用户唯一的全局身份标识，用于所有 Agent 的调用鉴权 |
| **Agent ID** | 平台内每个独立 Agent 的唯一标识 |
| **Endpoint** | 统一的 API 入口，兼容 OpenAI 协议格式，降低第三方工具集成成本 |

---

## 2. 功能模块 (Functional Modules)

### 2.1 Agent 市场 (Marketplace)

**功能描述：** 展示所有可用的 Agent 列表及详细参数。

**核心字段：**
- `Name` / `Description`: Agent 功能简述
- `Category`: 领域分类（开发 / 写作 / 创意 / 分析 / 其他）
- `Capability`: 核心能力标签（如：Code Review, 韩寒风格, 网文叙事, 剧本创作, 商业文案）
- `Context Window`: 支持的最大上下文长度
- `Pricing`: 调用成本（Credit/1K Tokens）

**交互逻辑：** 点击「雇佣/启用」后，该 Agent 进入用户的已激活列表。

### 2.2 集成网关 (Integration Gateway)

**协议兼容：** 必须完全兼容 OpenAI Chat Completions 接口标准。

**路由逻辑：**
- 系统通过请求头中的 `Authorization` 识别用户
- 通过 URL 路径或 Model 字段（如 `model: "asgard/architect-agent"`）路由至后端具体的 Agent 工程逻辑

**流式输出：** 全面支持 Server-Sent Events (SSE)，确保 IDE 端响应的实时性。

### 2.3 开发者控制台 (Developer Console)

- **凭证管理：** API Key 的创建、轮换与禁用
- **集成指南：** 针对主流工具（Cursor, VS Code (Continue), Raycast, Github Actions）提供配置模板
- **用量统计：** 实时展示各 Agent 的调用频次、Token 消耗及错误率

## 2.4 产品界面与控制台设计 (UI & Console Design)

### 2.4.1 整体信息架构

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航：Logo | Agent 市场 | 控制台 | 文档 | 用户/余额    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  主内容区（根据当前路由切换）                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

- **一级导航：** Agent 市场、开发者控制台、文档中心
- **全局入口：** 用户头像下拉（账户设置、退出）+ 余额/额度展示
- **快捷入口：** 顶部固定「获取 API Key」按钮，引导新用户完成首次配置

### 2.4.2 Agent 市场界面

**布局结构：**
- **左侧筛选区：** 领域分类（开发、写作、创意、分析等）、能力标签多选、上下文长度区间、价格区间
- **主列表区：** Agent 卡片网格，支持按「最新」「热度」「价格从低到高」排序
- **卡片内容：** 名称、一句话描述、能力标签、Context Window、Pricing、操作按钮「启用」

**交互规范：**
- 点击卡片进入详情页，展示完整参数说明与调用示例
- 已启用的 Agent 在卡片上显示「已启用」徽章，可一键停用
- 支持搜索框按名称或能力关键词检索

### 2.4.3 开发者控制台界面

**控制台首页：**
- **概览卡片：** 当前余额、今日/本周 Token 消耗、活跃 Agent 数量
- **快捷操作：** 创建 Key、查看集成指南、跳转用量明细

**凭证管理页：**
- **Key 列表表格：** 名称、标签（前几位）、创建时间、状态（启用/禁用）、操作（复制、轮换、禁用、删除）
- **新建 Key 弹窗：** 输入名称、可选设置额度上限、可选设置 IP/域名白名单
- **创建成功：** 仅一次展示完整 Key 字符串，并提示妥善保存；支持一键复制

**集成指南页：**
- **工具选项卡：** Cursor、VS Code、Notion、飞书、Figma、Raycast、Github Actions 等
- **每项内容：** 步骤说明 + 配置模板（可直接复制）+ 示例 Base URL 与 Model 字段

**用量统计页：**
- **时间维度：** 日/周/月切换
- **数据展示：** 按 Agent 维度展示调用次数、Input/Output Token、错误率折线图
- **导出：** 支持 CSV 导出，便于对账与成本分析

### 2.4.4 视觉与交互原则

- **风格：** 简洁、偏技术感，避免过度装饰；主色与品牌一致
- **响应式：** 控制台需在桌面端完整可用，移动端至少支持查看余额与用量
- **反馈：** 关键操作（创建 Key、禁用、轮换）需二次确认；成功/失败有明确 Toast 提示

## 3. 业务流程 (Workflow)

| 阶段 | 说明 |
|------|------|
| **配置阶段** | 用户在 Asgard 官网获取 API Key，并根据文档将其填入目标工具（如 Cursor、Notion、飞书、Figma 等） |
| **请求阶段** | 目标工具发起请求至 `https://api.asgard.ai/v1` |
| **处理阶段** | Asgard 网关层校验 Key 有效性及余额；网关根据请求指向的 `agent_id` 触发后端对应的 Agent 工程流（Agentic Workflow） |
| **返回阶段** | Agent 处理结果通过标准协议流式回传至目标工具 |

---

## 4. 技术约束与性能 (Technical Constraints)

| 维度 | 指标 |
|------|------|
| **延迟目标** | 网关层自身的路由损耗需控制在 50ms 以内 |
| **并发处理** | MVP 版本需支持单用户并发 5 个 Request Stream |
| **安全性** | 所有 API Key 必须进行单向哈希存储；支持基于 IP 或域名设置调用白名单 |

---

## 5. MVP 路线图 (Roadmap)

| 阶段 | 交付内容 |
|------|----------|
| **Phase 1** | 实现统一 API 网关 + 多领域 Agent（开发：代码重构/单元测试；写作：韩寒风格/网文叙事；创意：商业文案/剧本） |
| **Phase 2** | 增加 CLI 工具包，支持本地文件流直接与 Agent 交互 |
| **Phase 3** | 开放自定义配置，允许用户微调 Agent 的召回参数（Temperature, Top_p 等） |
